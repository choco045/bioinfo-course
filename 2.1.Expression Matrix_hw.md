## 1) 请阐述 RNA-seq 中归一化基因表达值的几种基本计算方法。

RNA-seq 原始 reads 计数（raw counts）受到测序深度、基因长度等因素的影响，不能直接用于样本间或基因间的比较，因此需要进行归一化（normalization）。以下是几种常用的归一化方法：

### 1. CPM / RPM（Counts Per Million）

**公式：**

$$\text{CPM} = \frac{\text{某基因的 mapped reads 数}}{\text{该样本总 mapped reads 数} / 10^6}$$

**特点：**
- 仅校正**测序深度**差异，不校正基因长度
- 适用于 **small RNA-seq**（miRNA 等长度相近，无需校正长度）
- 不适合用于比较不同长度基因之间的表达量

### 2. RPKM（Reads Per Kilobase per Million mapped reads）

**公式：**

$$\text{RPKM} = \frac{\text{某基因的 mapped reads 数}}{\text{总 mapped reads 数} / 10^6 \times \text{基因长度(Kbp)}}$$

**特点：**
- 同时校正**测序深度**和**基因长度**
- 适用于 **单端（single-end）** poly-A RNA-seq 或 total RNA-seq
- 缺点：不同样本的 RPKM 之和不相等，样本间可比性较差

### 3. FPKM（Fragments Per Kilobase per Million mapped reads）

**公式：**

$$\text{FPKM} = \frac{\text{RPKM}}{2}$$

**特点：**
- 是 RPKM 的双端测序版本
- 适用于 **双端（paired-end）** RNA-seq（每个 fragment 产生两条 reads）
- 同样存在样本间可比性差的问题

### 4. TPM（Transcripts Per Million）

**公式：**

$$\text{TPM} = \frac{\text{RPKM}}{\sum_{\text{所有基因}} \text{RPKM}} \times 10^6$$

**特点：**
- 同时校正**测序深度**和**基因长度**
- **所有样本的 TPM 之和均为 10⁶**，样本间可直接比较
- 是目前**最推荐**用于基因表达量比较的标准化方法
- 相比 RPKM/FPKM，TPM 更适合跨样本比较

### 5. TMM（Trimmed Mean of M-values，edgeR 使用）

**特点：**
- 专为**差异表达分析**设计，用于 edgeR 软件
- 计算步骤：
  1. 计算每个基因的 M 值（log2 倍数变化）和 A 值（平均表达量）
  2. 截去 M 值上下各 30%、A 值上下各 5%，得到代表性基因集
  3. 对代表性基因集计算加权 M 值均值，得到 TMM 因子
  4. 标准化：raw counts / 2^TMM
- 假设大多数基因在样本间不差异表达

### 6. RLE（Relative Log Expression，DESeq2 使用）

**特点：**
- 专为**差异表达分析**设计，用于 DESeq2 软件
- 计算步骤：
  1. 计算每个基因在所有样本中 raw counts 的几何均值
  2. 每个基因的 counts 除以其几何均值
  3. 取每个样本中所有比值的中位数作为标准化因子
  4. 标准化：raw counts / 标准化因子

## 4) 提供了结肠癌（COAD）、直肠癌（READ）和食道癌（ESCA）三种癌症各 50 个样本的 bam 文件用 featureCount 计算产生的结果。请编写脚本将这些文件中的 counts 合并到一个矩阵中（行为基因，列为样本），计算 logCPM 的 Z-score，并用 heatmap 展示，提供代码和 heatmap。根据 heatmap 可视化的结果，你认为这三种癌症中哪两种癌症的转录组是最相似的？

### 完整 R 代码

```r
library(edgeR)
library(pheatmap)

# ============================================================
# 步骤1：解压数据文件
# ============================================================

# 先在终端中执行：tar xvzf tumor-transcriptome-demo.tar.gz
# 解压后得到 COAD、READ、ESCA 三个文件夹，每个文件夹含50个 .txt 文件

# ============================================================
# 步骤2：读取并合并所有样本的 counts 数据
# ============================================================

# 获取所有样本文件路径
data_dir <- "tumor-transcriptome-demo"
cancer_types <- c("COAD", "READ", "ESCA")

all_files <- c()
for (cancer in cancer_types) {
  files <- list.files(file.path(data_dir, cancer), pattern = "\\.txt$", full.names = TRUE)
  all_files <- c(all_files, files)
}

cat("共找到", length(all_files), "个样本文件\n")

# 读取第一个文件，获取基因列表
first_data <- read.table(all_files[1], header = TRUE, skip = 1, sep = "\t", stringsAsFactors = FALSE)
gene_ids <- first_data[, 1]  # 第1列为 gene_id

# 初始化 counts 矩阵（行为基因，列为样本）
counts_matrix <- matrix(0, nrow = length(gene_ids), ncol = length(all_files))
rownames(counts_matrix) <- gene_ids
colnames(counts_matrix) <- paste0(basename(dirname(all_files)), "_", basename(all_files))

# 逐个读取样本文件，填充 counts 矩阵
for (i in seq_along(all_files)) {
  data <- read.table(all_files[i], header = TRUE, skip = 1, sep = "\t", stringsAsFactors = FALSE)
  counts_matrix[, i] <- data[, 7]  # 第7列为 counts
  if (i %% 10 == 0) cat("已处理", i, "/", length(all_files), "个文件\n")
}

cat("表达矩阵维度：", nrow(counts_matrix), "个基因 ×", 
    ncol(counts_matrix), "个样本\n")

# ============================================================
# 步骤3：计算 log10 CPM
# ============================================================

# 方法：使用 edgeR 的 cpm 函数
y <- DGEList(counts = counts_matrix)  # 创建 DGEList 对象

# 计算 CPM（不取对数）
CPM.matrix <- edgeR::cpm(y, log = FALSE)

# 计算 log10 CPM（加 pseudocount=1 避免 log(0)）
log10.CPM.matrix <- log10(CPM.matrix + 1)

# ============================================================
# 步骤4：计算 Z-score
# ============================================================

# Z-score = (某样本表达量 - 该基因平均表达量) / 该基因表达量标准差
# apply(matrix, 1, sd)：按行（1）计算标准差
z.scores <- (log10.CPM.matrix - rowMeans(log10.CPM.matrix)) / apply(log10.CPM.matrix, 1, sd)

# 过滤掉标准差为0的基因（所有样本表达量相同，Z-score 为 NaN）
z.scores <- z.scores[!is.nan(rowSums(z.scores)), ]
cat("过滤后保留", nrow(z.scores), "个基因\n")

# Clip Z-score：将 >2 的值截断为 2，<-2 的值截断为 -2
# 避免个别 outlier 导致大多数样本颜色差异不明显
z.scores.clipped <- z.scores
z.scores.clipped[z.scores.clipped > 2] <- 2
z.scores.clipped[z.scores.clipped < -2] <- -2

# ============================================================
# 步骤5：准备样本注释信息（用于热图颜色标注）
# ============================================================

# 创建样本注释数据框
sample_names <- colnames(z.scores.clipped)
# 根据文件名判断癌症类型（COAD/READ/ESCA）
cancer_annotation <- data.frame(
  Cancer_Type = ifelse(grepl("COAD", sample_names), "COAD",
                ifelse(grepl("READ", sample_names), "READ", "ESCA")),
  row.names = sample_names
)

# 设置注释颜色
annotation_colors <- list(
  Cancer_Type = c(COAD = "#E41A1C",   # 红色：结肠癌
                  READ = "#377EB8",   # 蓝色：直肠癌
                  ESCA = "#4DAF4A")   # 绿色：食道癌
)

# ============================================================
# 步骤6：绘制热图
# ============================================================

# 保存热图为 PDF 文件
pdf("tumor_transcriptome_heatmap.pdf", width = 14, height = 10)

pheatmap(
  z.scores.clipped,
  
  # 聚类设置
  cluster_rows = TRUE,          # 对基因（行）进行聚类
  cluster_cols = TRUE,          # 对样本（列）进行聚类
  clustering_method = "ward.D2", # 聚类方法
  
  # 颜色设置
  color = colorRampPalette(c("#2166AC", "white", "#D6604D"))(100),
  # 蓝色（低表达）→ 白色（中等）→ 红色（高表达）
  
  # 样本注释
  annotation_col = cancer_annotation,
  annotation_colors = annotation_colors,
  
  # 显示设置
  show_rownames = FALSE,        # 不显示基因名（基因太多）
  show_colnames = FALSE,        # 不显示样本名（样本太多）
  
  # 标题
  main = "Tumor Transcriptome Heatmap\n(log10 CPM Z-score, clipped to [-2, 2])",
  
  # 字体大小
  fontsize = 10,
  
  # 图例标题
  legend_breaks = c(-2, -1, 0, 1, 2),
  legend_labels = c("-2", "-1", "0", "1", "2")
)

dev.off()
cat("热图已保存为 tumor_transcriptome_heatmap.pdf\n")
```

### 结果分析与结论

**热图解读**：

通过对 150 个样本（COAD × 50、READ × 50、ESCA × 50）的 log10 CPM Z-score 进行层次聚类热图可视化，可以观察到：

1. **COAD（结肠癌）和 READ（直肠癌）的转录组最为相似**

   **原因分析**：
   - COAD（结肠癌）和 READ（直肠癌）在解剖学上都属于**大肠（colorectal）**，组织来源相近
   - 根据热图顶部的样本聚类树结果可以直观看出，结肠癌（COAD）和直肠癌（READ）的转录组是最相似的，它们在聚类时被紧密地分在了同一个大类群中。
   - 这与已知的生物学事实一致：结直肠癌（colorectal cancer, CRC）在分子分型上具有高度相似性，TCGA 等大型研究也证实了这一点

2. **ESCA（食道癌）与前两者差异较大**
   - 食道癌来源于消化道上段（食道），组织类型与结直肠不同
   - 在热图中，ESCA 样本会形成独立的聚类，与 COAD/READ 明显分离

**结论**：

- **COAD（结肠癌）和 READ（直肠癌）的转录组最为相似。**
- 这两种癌症均起源于大肠（结肠和直肠），具有相似的组织来源和分子特征，因此在基因表达谱上高度相似。相比之下，ESCA（食道癌）起源于食道上皮，与结直肠癌在转录组层面存在显著差异。
