## Input data: `homework_FemaleLiver-01-dataInput.RData`。Please try to construct an automatic network and detect module (choosing the soft-thresholding power, one-step network construction and module detection) from the `homework_FemaleLiver-01-dataInput.RData`. Please plot two figures: a. Analysis of network topology (Scale independence, Mean connectivity) for various soft-thresholding powers. b. The dendrogram and the module colors underneath.

本题可使用 WGCNA 对 134 个样本、3600 个基因的表达矩阵构建自动共表达网络。核心步骤是：读取表达矩阵、检查缺失值和异常样本、选择满足近似无尺度网络的软阈值、用 `blockwiseModules()` 一步法构建网络并识别模块，最后绘制软阈值诊断图和聚类树/模块颜色图。

```r
# 加载 WGCNA 包
library(WGCNA)

# 允许 WGCNA 使用多线程，加快相关性和 TOM 计算
enableWGCNAThreads()

# 读取作业数据；该 RData 通常会载入表达矩阵对象 datExpr 或类似对象
load("homework_FemaleLiver-01-dataInput.RData")

# 如果载入对象不是 datExpr，可用 ls() 查看对象名后手动赋值
ls()

# 假设表达矩阵对象名为 datExpr；行是样本，列是基因
datExpr <- as.data.frame(datExpr)

# 检查基因和样本是否存在过多缺失值或异常值
gsg <- goodSamplesGenes(datExpr, verbose = 3)

# 若存在不合格样本或基因，则过滤后再分析
if (!gsg$allOK) {
  datExpr <- datExpr[gsg$goodSamples, gsg$goodGenes]
}

# 对样本进行层次聚类，用于观察是否存在明显离群样本
sampleTree <- hclust(dist(datExpr), method = "average")
plot(sampleTree, main = "Sample clustering to detect outliers",
     xlab = "", sub = "", cex = 0.6)

# 设置待测试的软阈值范围
powers <- c(1:10, seq(12, 20, by = 2))

# 计算不同 power 下的无尺度拟合指数和平均连通性
sft <- pickSoftThreshold(datExpr, powerVector = powers, verbose = 5)

# 输出软阈值诊断图：左图为 Scale independence，右图为 Mean connectivity
pdf("FemaleLiver_soft_threshold_topology.pdf", width = 10, height = 5)
par(mfrow = c(1, 2))

# 绘制无尺度拓扑拟合指数；通常选择 signed R^2 首次达到 0.85 左右且连通性不过低的 power
plot(sft$fitIndices[, 1],
     -sign(sft$fitIndices[, 3]) * sft$fitIndices[, 2],
     xlab = "Soft Threshold (power)",
     ylab = "Scale Free Topology Model Fit, signed R^2",
     type = "n",
     main = "Scale independence")
text(sft$fitIndices[, 1],
     -sign(sft$fitIndices[, 3]) * sft$fitIndices[, 2],
     labels = powers,
     col = "red")
abline(h = 0.85, col = "red")

# 绘制不同 power 下的平均连通性，用于避免 power 过大导致网络过稀
plot(sft$fitIndices[, 1],
     sft$fitIndices[, 5],
     xlab = "Soft Threshold (power)",
     ylab = "Mean Connectivity",
     type = "n",
     main = "Mean connectivity")
text(sft$fitIndices[, 1],
     sft$fitIndices[, 5],
     labels = powers,
     col = "red")
dev.off()

# 选择软阈值：若 pickSoftThreshold 已给出 powerEstimate，则优先使用；
# 否则选择 signed R^2 接近或超过 0.85 且平均连通性仍合理的 power，例如 6
softPower <- sft$powerEstimate
if (is.na(softPower)) {
  softPower <- 6
}

# 一步法构建加权共表达网络并识别模块
net <- blockwiseModules(
  datExpr,
  power = softPower,
  TOMType = "unsigned",
  minModuleSize = 30,
  reassignThreshold = 0,
  mergeCutHeight = 0.25,
  numericLabels = TRUE,
  pamRespectsDendro = FALSE,
  saveTOMs = TRUE,
  saveTOMFileBase = "FemaleLiver-TOM",
  verbose = 3
)

# 将数字模块标签转换为颜色标签，便于可视化
moduleColors <- labels2colors(net$colors)

# 输出模块颜色统计
table(moduleColors)

# 绘制基因聚类树和对应模块颜色
pdf("FemaleLiver_gene_dendrogram_module_colors.pdf", width = 12, height = 8)
plotDendroAndColors(
  net$dendrograms[[1]],
  moduleColors[net$blockGenes[[1]]],
  "Module colors",
  dendroLabels = FALSE,
  hang = 0.03,
  addGuide = TRUE,
  guideHang = 0.05,
  main = "Gene dendrogram and module colors"
)
dev.off()

# 保存主要结果，方便后续提取模块基因或做性状相关分析
save(datExpr, net, moduleColors, softPower,
     file = "FemaleLiver_WGCNA_modules.RData")
```

本题应提交或展示的结果文件包括：

- `FemaleLiver_soft_threshold_topology.pdf`：不同 soft-thresholding powers 下的 Scale independence 和 Mean connectivity。
- `FemaleLiver_gene_dendrogram_module_colors.pdf`：基因聚类树及其下方模块颜色。
- `FemaleLiver_WGCNA_modules.RData`：保存表达矩阵、模块检测结果、模块颜色和所选 soft power，便于后续复查。

## 根据提供的 miRNA 序列和 target sequence 序列，使用 miRanda 方法预测 miRNA 在这些 target sequence 上的靶标位点。

使用 miRanda 预测靶标时，需要两个 FASTA 文件：一个为 miRNA 序列，一个为候选 target sequence 序列。miRanda 会根据 miRNA 与靶序列的互补配对、结合能等信息输出预测的靶位点。

```bash
# 进入 miRNA target prediction 作业文件所在目录
cd /home/test/mirna

# 使用 miRanda 预测 miRNA 在 target sequence 上的结合位点
# 第一个 fasta 文件是 miRNA 序列
# 第二个 fasta 文件是候选靶序列
# 结果重定向保存到 miRanda.output.txt
miranda miRanda.miRNA.fa miRanda.target_sequence.fa > miRanda.output.txt

# 提取 miRanda 输出中的 Scores for this hit 结果行，便于整理成表格
# 这些行包含 miRNA、target、score、energy、alignment 起止位置等信息
grep -A 1 "Scores for this hit" miRanda.output.txt > miRanda.hit.summary.txt
```

miRanda 输出中每个预测靶位点主要包含以下信息：

| 字段 | 含义 |
|------|------|
| `Score` | miRNA 与 target 匹配的综合得分，分数越高表示序列互补性越好 |
| `Energy` | miRNA-target duplex 的自由能，数值越负表示结合越稳定 |
| `Q:start-end` | miRNA 上参与比对的区间 |
| `R:start-end` | target sequence 上预测靶位点的位置 |
| `Align Len` | miRNA 与 target 的比对长度 |
| `Subject-Identity` | target 侧匹配比例 |
| `Query-Identity` | miRNA 侧匹配比例 |

整理结果时保留 `miRanda.output.txt` 作为完整预测结果，并将命中位点汇总为 `miRanda.hit.summary.txt`。判断较可信靶位点时，优先关注 score 较高、energy 更负、seed 区互补性较好的预测结果。

## 根据上述代码复现 CLIP-seq 的分析过程，筛选出所有 FDR < 0.05 的 CIMS，并提交对应的文件。

本题使用 CTK 对 PAR-CLIP 数据进行 CIMS 分析。分析思路是：先从比对结果中解析 read 位点和突变信息，再去除 PCR duplication，随后提取 T-to-C conversion，运行 `CIMS.pl` 识别 cross-link-induced mutation sites，最后按第 9 列 FDR 筛选显著 CIMS。

```bash
# 进入 CLIP-seq 工作目录
mkdir -p /home/test/clip
cd /home/test/clip

# 对两个样本循环分析
for i in SRR048967 SRR048968
do
  echo ${i}

  # 从 sam.gz 比对结果中解析 reads 的基因组位置，并同时输出突变信息
  # --map-qual 1 要求最低 mapping quality 为 1
  # --min-len 18 只保留长度至少为 18 nt 的 reads
  # --mutation-file 保存 read 与参考基因组相比的突变信息
  perl /home/test/app/ctk/parseAlignment.pl \
    -v \
    --map-qual 1 \
    --min-len 18 \
    --mutation-file ${i}.mutations.txt \
    /home/test/file/input/${i}.sam.gz \
    ${i}.tag.bed

  # 合并 PCR duplication；CTK 会把基因组起始位置相同的 reads 视为重复
  # -weight 和 --weight-in-name 表示合并时考虑 read 权重
  # --keep-max-score 保留权重最高的 read
  perl /home/test/app/ctk/tag2collapse.pl \
    -v \
    -big \
    -weight \
    --weight-in-name \
    --keep-max-score \
    --keep-tag-name \
    ${i}.tag.bed \
    ${i}.tag.uniq.bed

  # 从全部 mutation 中保留去重后 read 对应的 mutation 信息
  # 两个 4 表示用两个文件的第 4 列 read ID 进行匹配
  python2 /home/test/app/ctk/joinWrapper.py \
    ${i}.mutations.txt \
    ${i}.tag.uniq.bed \
    4 \
    4 \
    N \
    ${i}.tag.uniq.mutations.txt

  # 提取 PAR-CLIP 中最重要的 T-to-C conversion
  # --summary 输出各类 mutation 的统计结果
  perl /home/test/app/ctk/getMutationType.pl \
    -t t2c \
    --summary ${i}.tag.uniq.mutations.stat.txt \
    ${i}.tag.uniq.mutations.txt \
    ${i}.tag.uniq.t2c.bed

  # 运行 CIMS 分析，识别可能的 RNA-RBP cross-linking 位点
  # 输入为去重后的 read 位点文件和 T-to-C mutation 文件
  # 输出 ${i}.tag.uniq.t2c.CIMS.txt，其中第 9 列为 FDR
  perl /home/test/app/ctk/CIMS.pl \
    -big \
    -n 10 \
    -p \
    -outp ${i}.tag.uniq.t2c.pos.stat.txt \
    -v \
    ${i}.tag.uniq.bed \
    ${i}.tag.uniq.t2c.bed \
    ${i}.tag.uniq.t2c.CIMS.txt

  # 筛选 FDR <= 0.05 的显著 CIMS
  # 先按 FDR 升序，再按第 8 列降序、第 7 列升序排序，便于查看高置信位点
  awk '{if($9<=0.05) {print $0}}' ${i}.tag.uniq.t2c.CIMS.txt \
    | sort -k 9,9n -k 8,8nr -k 7,7n \
    > ${i}.tag.uniq.t2c.CIMS.s13.txt
done

# 可选：合并两个样本的显著 CIMS，方便统一提交和查看
cat SRR048967.tag.uniq.t2c.CIMS.s13.txt \
    SRR048968.tag.uniq.t2c.CIMS.s13.txt \
    > all_samples.FDR_lt_0.05.CIMS.txt
```

- `SRR048967.tag.uniq.t2c.CIMS.s13.txt`
- `SRR048968.tag.uniq.t2c.CIMS.s13.txt`
- `all_samples.FDR_lt_0.05.CIMS.txt`（可选合并文件）

这些文件中的每一行代表一个通过 FDR 阈值筛选的显著 cross-link-induced mutation site，其中第 9 列满足 `FDR <= 0.05`。

## 查阅资料回答，在 ChIP-seq 中是不是和 CLIP-seq 一样也存在 PCR duplication 导致假阳性 peak 的情况？如果有，HOMER 和 MACS2 这类工具是怎么处理这一问题的？

ChIP-seq 中同样存在 PCR duplication 导致假阳性 peak 的情况。文库制备时，如果少数 DNA 片段在 PCR 扩增中被过度扩增，测序后会表现为大量 reads 堆叠在完全相同或几乎相同的基因组位置。若不处理，这些重复 reads 会被误认为该区域有很强的蛋白结合或组蛋白修饰信号，从而造成假阳性 peak，尤其在测序深度较高、起始 DNA 量较低或 PCR cycle 过多时更明显。

HOMER 和 MACS2 都会在 peak calling 中处理这个问题，但策略略有不同。

**HOMER 的处理方式：**

HOMER 在构建 tag directory 时会记录 reads 的位置和方向，并可识别同一位置上的过度重复 tags。后续 `findPeaks` 会使用 clonal tag 过滤思想，即限制同一基因组位置允许保留的重复 tag 数量。`findPeaks` 中的 `-C <#>` 参数表示 clonal fold change 阈值，用于过滤可能由 clonality/PCR duplication 造成的异常富集区域。若某一区域由少数完全重复 reads 支撑，而不是由分布合理的独立片段支撑，HOMER 会降低其被判为真实 peak 的可能性。

**MACS2 的处理方式：**

MACS2 使用 `--keep-dup` 参数控制重复 reads 的保留方式。常用设置包括：

```bash
# 默认模式：MACS2 根据二项分布模型自动估计每个位置可接受的重复 reads 数
macs2 callpeak -t chip.bam -c input.bam -f BAM -g hs -n sample

# 只保留每个位置 1 条重复 read，过滤最严格
macs2 callpeak -t chip.bam -c input.bam -f BAM -g hs -n sample_keep1 --keep-dup 1

# 保留所有重复 reads；一般不推荐，除非数据已用 UMI 或其他方法去重
macs2 callpeak -t chip.bam -c input.bam -f BAM -g hs -n sample_all --keep-dup all
```

MACS2 默认会对同一位置的重复 reads 进行限制，而不是无条件全部保留。这样可以减少 PCR duplication 对局部 read pileup 的影响。对于真正的强结合位点，reads 通常会在结合中心附近形成有方向、有宽度的富集分布；而 PCR duplication 更倾向于表现为同一坐标的异常堆叠，因此去重或限制重复数可以有效降低假阳性。

综上，ChIP-seq 和 CLIP-seq 都需要关注 PCR duplication。CLIP-seq 中 CTK 的 `tag2collapse.pl` 会按相同起始位置合并重复 reads；ChIP-seq 中 HOMER 通过 clonal tag 过滤和 `-C` 参数控制重复信号，MACS2 通过 `--keep-dup` 控制同一位置重复 reads 的保留数量。
