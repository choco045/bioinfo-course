## 什么是 Multiple test correction？并解释 q value（很多时候也叫 FDR）和 p value 的差别。

### 什么是 Multiple test correction（多重检验校正）？

在 RNA-seq 差异表达分析中，我们通常需要同时对成千上万个基因进行假设检验（每个基因做一次 t 检验或类似检验）。如果我们对每个基因都使用显著性阈值 α = 0.05，那么即使所有基因都没有真实差异，仅凭随机误差，我们也会期望有 5% 的基因被错误地判断为显著（即假阳性）。

**举例说明：**
- 假设我们检验 20,000 个基因
- 即使所有基因都不差异表达，仅凭随机误差，预期会有 20,000 × 0.05 = **1,000 个假阳性**
- 这种现象称为**多重检验问题（Multiple Testing Problem）**

**Multiple test correction（多重检验校正）** 是一类统计方法，用于在同时进行多个假设检验时控制假阳性率，使整体错误率保持在可接受的水平。

常见的校正方法包括：

| 方法 | 控制目标 | 特点 |
|------|----------|------|
| **Bonferroni 校正** | 家族错误率（FWER） | 最保守，将 α 除以检验次数 n，即 α' = α/n |
| **Benjamini-Hochberg（BH）** | 错误发现率（FDR） | 最常用，比 Bonferroni 更宽松，统计功效更高 |
| **Storey's q value** | 正期望错误发现率（pFDR） | 基于 BH 的改进，考虑了真实零假设的比例 π₀ |

在 RNA-seq 分析中，DESeq2 和 edgeR 默认使用 **Benjamini-Hochberg（BH）方法**进行多重检验校正。

---

### p value 与 q value（FDR）的区别

#### p value（p 值）

**p value** 是在**零假设（H₀）为真**的条件下，观测到当前统计量或更极端结果的概率。

$$p = P(\text{观测到的统计量} \geq T \mid H_0 \text{ 为真})$$

- p value 是**单个检验**的指标
- p value 小说明在零假设下观测到该结果的概率很低，即有证据拒绝零假设
- **不考虑**同时进行多少个检验

#### q value / FDR（错误发现率）

**FDR（False Discovery Rate，错误发现率）** 是在所有被判断为显著的结果中，假阳性所占的比例的期望值：

$$\text{FDR} = E\left[\frac{\text{假阳性数（FP）}}{\text{所有显著结果数（FP + TP）}}\right]$$

**q value** 是对每个基因计算的 FDR 估计值，表示：如果以该基因的 p value 为阈值，在所有被判断为显著的基因中，预期假阳性的比例。

$$q_i = \frac{\pi_0 \cdot m \cdot p_i}{\text{rank}(p_i)}$$

其中：
- $\pi_0$：真实零假设的比例（即真正不差异表达的基因比例）
- $m$：总检验次数
- $p_i$：第 $i$ 个基因的 p value
- $\text{rank}(p_i)$：$p_i$ 在所有 p value 中的排名

#### 核心区别总结

| 特性 | p value | q value / FDR |
|------|---------|---------------|
| **定义** | 单个检验中，H₀ 为真时观测到当前结果的概率 | 所有显著结果中假阳性的期望比例 |
| **考虑范围** | 单个基因 | 所有基因（全局） |
| **多重检验** | 不考虑 | 专门用于校正多重检验 |
| **阈值含义** | p < 0.05 表示单次检验假阳性率 < 5% | FDR < 0.05 表示显著基因中假阳性 < 5% |
| **统计功效** | 高（但假阳性多） | 较高（在控制假阳性的同时保留更多真阳性） |
| **在 DESeq2 中** | `pvalue` 列 | `padj` 列 |
| **在 edgeR 中** | `PValue` 列 | `FDR` 列 |

**实际意义举例：**
- 若某基因 p value = 0.001，FDR = 0.05：
  - p value 说明：在该基因不差异表达的前提下，观测到这么大差异的概率只有 0.1%
  - FDR 说明：在所有 FDR ≤ 0.05 的基因中，预期有 5% 是假阳性

## 请结合上课时所讲的知识阐述 DESeq2 和 edgeR 中如何对数据进行 normalization，列出并解释具体的公式。

### 为什么需要 Normalization？

在 RNA-seq 实验中，不同样本之间的原始 read count 不能直接比较，原因包括：
1. **测序深度（Library size）不同**：测序量越大，每个基因的 count 越高
2. **基因长度不同**：较长的基因会产生更多 reads（但 DESeq2/edgeR 处理的是 count 矩阵，已经是 exon-level 的 count，这点影响较小）
3. **样本间 RNA 组成差异**：少数高表达基因可能占据大量 reads，影响其他基因的相对比例

---

### DESeq2 的 Normalization：Median of Ratios 方法

DESeq2 使用 **Median of Ratios（中位数比值法）** 进行归一化，通过 `estimateSizeFactors()` 函数实现。

#### 具体步骤与公式

**Step 1：计算每个基因在所有样本中的几何平均值（参考样本）**

$$\bar{K}_j = \left(\prod_{i=1}^{n} K_{ij}\right)^{1/n} = \exp\left(\frac{1}{n}\sum_{i=1}^{n} \ln K_{ij}\right)$$

其中：
- $K_{ij}$：基因 $j$ 在样本 $i$ 中的原始 count
- $n$：样本数
- $\bar{K}_j$：基因 $j$ 的几何平均 count（作为"参考样本"）

**Step 2：计算每个样本中每个基因相对于参考样本的比值**

$$r_{ij} = \frac{K_{ij}}{\bar{K}_j}$$

**Step 3：取每个样本所有基因比值的中位数，作为该样本的 size factor**

$$s_i = \text{median}_j\left(\frac{K_{ij}}{\bar{K}_j}\right)$$

> 注意：几何平均值为 0 的基因（即在任意样本中 count = 0 的基因）会被排除在计算之外。

**Step 4：用 size factor 对原始 count 进行归一化**

$$\tilde{K}_{ij} = \frac{K_{ij}}{s_i}$$

#### 为什么用中位数而不是均值？

使用**中位数**而非均值，是为了对少数极高表达基因（如管家基因）的影响具有鲁棒性。这些基因的比值可能极大，但中位数不受极端值影响。

#### DESeq2 的统计模型

归一化后，DESeq2 对每个基因 $j$ 在样本 $i$ 中的 count 建立负二项分布模型：

$$K_{ij} \sim \text{NB}(\mu_{ij}, \alpha_j)$$

$$\mu_{ij} = s_i \cdot q_{ij}$$

$$\ln(q_{ij}) = \beta_{j0} + \beta_{j1} \cdot x_i$$

其中：
- $\mu_{ij}$：期望 count
- $\alpha_j$：基因 $j$ 的离散度参数（dispersion）
- $s_i$：样本 $i$ 的 size factor
- $q_{ij}$：归一化后的期望表达量
- $\beta_{j0}$：截距（基础表达水平）
- $\beta_{j1}$：处理效应（即 log2FoldChange）
- $x_i$：样本 $i$ 的处理条件（0 = Control，1 = Treatment）

---

### edgeR 的 Normalization：TMM 方法

edgeR 使用 **TMM（Trimmed Mean of M-values，M 值的截尾均值）** 方法进行归一化，通过 `calcNormFactors()` 函数实现。

#### 核心思想

TMM 方法基于以下假设：**大多数基因在不同条件下不差异表达**，因此可以用这些基因来估计样本间的系统性偏差。

#### 具体步骤与公式

**Step 1：选择参考样本**

选择 75th 百分位数 count 最接近所有样本平均值的样本作为参考样本 $r$。

**Step 2：计算 M 值（log fold change）和 A 值（平均表达量）**

对于样本 $k$ 中的基因 $g$，相对于参考样本 $r$：

$$M_g^{(k)} = \log_2\frac{Y_{gk}/N_k}{Y_{gr}/N_r}$$

$$A_g^{(k)} = \frac{1}{2}\log_2\left(Y_{gk}/N_k \cdot Y_{gr}/N_r\right)$$

其中：
- $Y_{gk}$：基因 $g$ 在样本 $k$ 中的原始 count
- $N_k$：样本 $k$ 的总 count（library size）
- $M_g^{(k)}$：log₂ fold change（M 值）
- $A_g^{(k)}$：平均 log₂ 表达量（A 值）

**Step 3：截尾（Trimming）**

去除 M 值最高和最低各 30% 的基因，以及 A 值最低 5% 的基因（低表达基因），避免极端值的影响。

**Step 4：计算加权均值作为 TMM 因子**

$$\log_2(f_k^{TMM}) = \frac{\sum_{g \in G^*} w_g^{(k)} M_g^{(k)}}{\sum_{g \in G^*} w_g^{(k)}}$$

其中权重 $w_g^{(k)}$ 为：

$$w_g^{(k)} = \frac{N_k - Y_{gk}}{N_k \cdot Y_{gk}} + \frac{N_r - Y_{gr}}{N_r \cdot Y_{gr}}$$

（权重与方差成反比，高表达基因权重更大）

**Step 5：归一化**

$$\text{有效 library size}_k = N_k \times f_k^{TMM}$$

归一化后的 CPM（Counts Per Million）：

$$\text{CPM}_{gk} = \frac{Y_{gk}}{N_k \times f_k^{TMM}} \times 10^6$$

#### edgeR 的统计模型

edgeR 同样使用负二项分布：

$$Y_{gk} \sim \text{NB}(\mu_{gk}, \phi_g)$$

$$\mu_{gk} = N_k \times f_k^{TMM} \times \lambda_{gk}$$

$$\ln(\lambda_{gk}) = \mathbf{x}_k^T \boldsymbol{\beta}_g$$

其中：
- $\phi_g$：基因 $g$ 的离散度（dispersion）
- $\lambda_{gk}$：归一化后的期望表达量
- $\mathbf{x}_k$：样本 $k$ 的设计向量
- $\boldsymbol{\beta}_g$：回归系数向量（包含 log2FoldChange）

---

### DESeq2 vs edgeR Normalization 对比

| 特性 | DESeq2（Median of Ratios） | edgeR（TMM） |
|------|---------------------------|--------------|
| **核心思想** | 用几何平均作为参考，取比值中位数 | 用截尾均值消除极端基因影响 |
| **鲁棒性** | 对极端高表达基因鲁棒（中位数） | 对极端高表达基因鲁棒（截尾） |
| **假设** | 大多数基因不差异表达 | 大多数基因不差异表达 |
| **输出** | size factor（每个样本一个值） | normalization factor（每个样本一个值） |
| **实际效果** | 两种方法结果通常非常接近 | 两种方法结果通常非常接近 |

## 利用以上介绍的方法和数据，分别使用 DESeq2 和 edgeR 找出 uvr8 突变型（uvr8\*）在光照前后的差异基因，保存为文本文件

### 分析思路

本题要求分析 **uvr8 突变型（uvr8\*）在光照前后**的差异基因，即：
- **Control**：uvr8\* 光照前（dark），对应数据第 8–10 列（`UVR8_0_1`, `UVR8_0_2`, `UVR8_0_3`）
- **Treatment**：uvr8\* 光照后（light），对应数据第 11–13 列（`UVR8_1_1`, `UVR8_1_2`, `UVR8_1_3`）

> ⚠️ 注意：列名需根据实际数据文件确认，以下代码中使用的列名为示例，请根据 `count_exon.txt` 的实际列名调整。

### 使用 DESeq2 分析 uvr8\* 差异基因

```r
library(DESeq2)

# Step 1: 读取原始 count 矩阵
raw.counts <- read.table("count_exon.txt", sep='\t', header = T, row.names = 1)

# Step 2: 提取 uvr8* 突变型数据（第 8-13 列）
# 注意：列名需根据实际数据文件确认
# 假设列名为 UVR8_0_1/2/3（光照前）和 UVR8_1_1/2/3（光照后）
uvr8.raw.counts <- raw.counts[, c("UVR8_0_1", "UVR8_0_2", "UVR8_0_3",
                                   "UVR8_1_1", "UVR8_1_2", "UVR8_1_3")]

# Step 3: 过滤低表达基因
uvr8.filtered.counts <- uvr8.raw.counts[rowMeans(uvr8.raw.counts) > 5, ]

# Step 4: 定义样本条件
# Control = 光照前（dark），Treatment = 光照后（light）
conditions <- factor(c(rep("Control", 3), rep("Treatment", 3)),
                     levels = c("Control", "Treatment"))
colData <- data.frame(row.names = colnames(uvr8.filtered.counts),
                      conditions = conditions)

# Step 5: 创建 DESeqDataSet 并进行差异分析
dds <- DESeqDataSetFromMatrix(uvr8.filtered.counts, colData, design = ~conditions)
dds <- DESeq(dds)
res <- results(dds)

# Step 6: 保存所有基因结果
write.table(res, "deseq2.uvr8.light.vs.dark.all.txt",
            sep='\t', row.names = T, quote = F)

# Step 7: 筛选差异显著基因（|log2FC| > 1, padj < 0.05）
diff.deseq2 <- subset(res, padj < 0.05 & abs(log2FoldChange) > 1)
write.table(diff.deseq2, "deseq2.uvr8.light.vs.dark.sig.txt",
            sep='\t', row.names = T, quote = F)

cat("DESeq2 找到的差异基因数：", nrow(diff.deseq2), "\n")
```

### 使用 edgeR 分析 uvr8\* 差异基因

```r
library(edgeR)

# Step 1: 读取数据（如果已读取可跳过）
raw.counts <- read.table("count_exon.txt", sep='\t', header = T, row.names = 1)

# Step 2: 提取 uvr8* 突变型数据
uvr8.raw.counts <- raw.counts[, c("UVR8_0_1", "UVR8_0_2", "UVR8_0_3",
                                   "UVR8_1_1", "UVR8_1_2", "UVR8_1_3")]

# Step 3: 过滤低表达基因
uvr8.filtered.counts <- uvr8.raw.counts[rowMeans(uvr8.raw.counts) > 5, ]

# Step 4: 定义样本条件和 design 矩阵
conditions <- factor(c(rep("Control", 3), rep("Treatment", 3)),
                     levels = c("Control", "Treatment"))
design <- model.matrix(~conditions)

# Step 5: 创建 DGEList 对象并进行 TMM 标准化
y <- DGEList(counts = uvr8.filtered.counts)
y <- calcNormFactors(y, method = "TMM")

# Step 6: 估计离散度并拟合 GLM
y <- estimateDisp(y, design = design)
fit <- glmFit(y, design = design)

# Step 7: 似然比检验
lrt <- glmLRT(fit, coef = 2)

# Step 8: 获取所有基因结果并保存
diff.table <- topTags(lrt, n = nrow(y))$table
write.table(diff.table, file = 'edger.uvr8.light.vs.dark.all.txt',
            sep = "\t", quote = F, row.names = T, col.names = T)

# Step 9: 筛选差异显著基因（|log2FC| > 1, FDR < 0.05）
diff.edger <- diff.table[abs(diff.table$logFC) > 1 & diff.table$FDR < 0.05, ]
write.table(diff.edger, file = 'edger.uvr8.light.vs.dark.sig.txt',
            sep = "\t", quote = F, row.names = T, col.names = T)

cat("edgeR 找到的差异基因数：", nrow(diff.edger), "\n")
```

### 结果解读

两个软件结果的差异通常来源于：

1. **归一化方法不同**：DESeq2 使用 Median of Ratios，edgeR 使用 TMM，对于某些样本可能产生不同的归一化因子
2. **离散度估计策略不同**：DESeq2 使用收缩估计（shrinkage），edgeR 使用 empirical Bayes 方法
3. **检验方法不同**：DESeq2 默认使用 Wald test，edgeR 使用似然比检验（LRT）

一般来说，两个软件的结果**高度一致**，共同找到的基因可信度更高。

## 对于 edgeR 找出的 FDR < 0.05 的基因，选出 log2FoldChange 最大的 10 个基因和最小的 10 个基因。计算原始表达量矩阵的 log10CPM 值并对每个基因进行 Z-score 处理，使用筛选出来的 20 个基因绘制热图（heatmap）作为最后结果输出。

### 分析思路

1. 从 edgeR 结果中筛选 FDR < 0.05 的基因
2. 按 logFC 排序，取最大的 10 个和最小的 10 个（共 20 个基因）
3. 计算这 20 个基因在所有样本中的 log10CPM 值
4. 对每个基因进行 Z-score 标准化
5. 绘制热图

### R 代码实现

```r
library(edgeR)
library(pheatmap)
library(RColorBrewer)

# Step 1: 获取 FDR < 0.05 的基因（使用上一题的 edgeR 结果）
# diff.table 包含所有基因的结果，已按 p 值排序
sig.genes <- diff.table[diff.table$FDR < 0.05, ]
cat("FDR < 0.05 的基因数：", nrow(sig.genes), "\n")

# Step 2: 按 logFC 排序，选出最大的 10 个和最小的 10 个
sig.genes.sorted <- sig.genes[order(sig.genes$logFC, decreasing = TRUE), ]

# 最大 logFC 的 10 个基因（上调最显著）
top10.up <- head(sig.genes.sorted, 10)
# 最小 logFC 的 10 个基因（下调最显著）
top10.down <- tail(sig.genes.sorted, 10)

# 合并 20 个基因
top20.genes <- c(rownames(top10.up), rownames(top10.down))
cat("选出的 20 个基因：\n")
cat("上调 Top 10：", rownames(top10.up), "\n")
cat("下调 Top 10：", rownames(top10.down), "\n")

# Step 3: 计算 log10CPM 值
# 使用 edgeR 的 cpm() 函数计算 CPM，然后取 log10
# y 是之前创建的 DGEList 对象（包含 TMM 归一化因子）
cpm.matrix <- cpm(y, normalized.lib.sizes = TRUE, log = FALSE)

# 提取 20 个基因的 CPM 矩阵
cpm.top20 <- cpm.matrix[top20.genes, ]

# 计算 log10CPM（加 1 避免 log(0)）
log10cpm.top20 <- log10(cpm.top20 + 1)

# Step 4: Z-score 标准化（对每个基因，在所有样本间进行标准化）
# scale() 函数默认对列进行标准化，所以需要先转置
zscore.top20 <- t(scale(t(log10cpm.top20)))

# 检查是否有 NA（如果某基因在所有样本中表达量相同，标准差为 0）
zscore.top20[is.nan(zscore.top20)] <- 0
```

### 结果解读

**热图说明：**
- **行**：20 个差异基因（上调 Top 10 + 下调 Top 10）
- **列**：6 个样本（3 个 dark/Control + 3 个 light/Treatment）
- **颜色**：红色表示高于平均表达（Z-score > 0），蓝色表示低于平均表达（Z-score < 0）
- **聚类**：行和列均进行层次聚类，相似表达模式的基因/样本会聚在一起

**预期结果：**
- 上调基因（logFC > 0）：在 light（Treatment）样本中颜色偏红，在 dark（Control）样本中颜色偏蓝
- 下调基因（logFC < 0）：在 light（Treatment）样本中颜色偏蓝，在 dark（Control）样本中颜色偏红
- 同一条件的 3 个生物学重复应该聚在一起，表明实验重复性良好

**Z-score 标准化的意义：**
- Z-score = (基因在某样本的 log10CPM - 该基因所有样本的均值) / 该基因所有样本的标准差
- 标准化后，每个基因的表达值以均值为中心，便于比较不同基因之间的相对变化模式
- 消除了不同基因绝对表达量差异的影响，突出了各基因在不同条件下的相对变化趋势
