## （1）什么是 Multiple test correction？并解释 q value（很多时候也叫 FDR）和 p value 的差别。

### Multiple test correction（多重检验校正）：

- 定义：在同一数据集上同时进行大量统计假设检验时，单次检验的假阳性错误（Type I error）会发生累积，导致全局的总体假阳性率（Family-Wise Error Rate, FWER）急剧上升。多重检验校正旨在通过数学算法（如 Bonferroni 或 Benjamini-Hochberg 法）重新调整显著性判定阈值，以严格控制整体的假阳性风险。对多次重复统计检验获得的概率进行重新计算，其本质是将原始的 P 值 (p value) 转换为调整后的 P 值 (adjusted p value)，也就是 Q 值 (q value)。

- 在 RNA-seq 差异表达分析中，我们通常需要同时对成千上万个基因进行假设检验（每个基因做一次 t 检验或类似检验）。  
  - 假设我们检验 20,000 个基因
  - 即使所有基因都不差异表达，仅凭随机误差，预期会有 20,000 × 0.05 = **1,000 个假阳性**
  - 这种现象称为**多重检验问题（Multiple Testing Problem）**

- **Multiple test correction** 是一类统计方法，用于在同时进行多个假设检验时控制假阳性率，使整体错误率保持在可接受的水平。

- 常见的校正方法包括：

  | 方法 | 控制目标 | 特点 |
  |------|----------|------|
  | **Bonferroni 校正** | 家族错误率（FWER） | 最保守，将 α 除以检验次数 n，即 α' = α/n |
  | **Benjamini-Hochberg（BH）** | 错误发现率（FDR） | 最常用，比 Bonferroni 更宽松，统计功效更高 |
  | **Storey's q value** | 正期望错误发现率（pFDR） | 基于 BH 的改进，考虑了真实零假设的比例 π₀ |

  在 RNA-seq 分析中，DESeq2 和 edgeR 默认使用 **Benjamini-Hochberg（BH）方法**进行多重检验校正。

### p value 与 q value（FDR）的区别

- **p value（p 值）**

  **p value** 是在**零假设（H₀）为真**的条件下，观测到当前统计量或更极端结果的概率。

  $$p = P(\text{观测到的统计量} \geq T \mid H_0 \text{ 为真})$$

  - p value 是**单个检验**的指标
  - p value 小说明在零假设下观测到该结果的概率很低，即有证据拒绝零假设
  - **不考虑**同时进行多少个检验

- **q value / FDR（错误发现率）**

  **FDR（False Discovery Rate，错误发现率）** 是在所有被判断为显著的结果中，假阳性所占的比例的期望值：

  $$\text{FDR} = E\left[\frac{\text{假阳性数（FP）}}{\text{所有显著结果数（FP + TP）}}\right]$$

  **q value** 是对每个基因计算的 FDR 估计值，表示：如果以该基因的 p value 为阈值，在所有被判断为显著的基因中，预期假阳性的比例。

  $$q_i = \frac{\pi_0 \cdot m \cdot p_i}{\text{rank}(p_i)}$$

  其中：
  - $\pi_0$：真实零假设的比例（即真正不差异表达的基因比例）
  - $m$：总检验次数
  - $p_i$：第 $i$ 个基因的 p value
  - $\text{rank}(p_i)$： $p_i$ 在所有 p value 中的排名

#### 核心区别总结

| 特性 | p value | q value / FDR |
|------|---------|---------------|
| **定义** | 单个检验中，H₀ 为真时观测到当前结果的概率 | 所有显著结果中假阳性的期望比例 |
| **考虑范围** | 单个基因 | 所有基因（全局） |
| **多重检验** | 不考虑 | 专门用于校正多重检验 |
| **阈值含义** | p < 0.05 表示单次检验假阳性率 < 5% | FDR < 0.05 表示显著基因中假阳性 < 5% |
| **统计功效** | 高（但假阳性多） | 较高（在控制假阳性的同时保留更多真阳性） |
| **在 DESeq2 中** | `pvalue` 列 | `padj` 列 |
| **在 edgeR 中** | `PValue` 列 | `FDR` 列 |

**实际意义举例：**
- 若某基因 p value = 0.001，FDR = 0.05：
  - p value 说明：在该基因不差异表达的前提下，观测到这么大差异的概率只有 0.1%
  - FDR 说明：在所有 FDR ≤ 0.05 的基因中，预期有 5% 是假阳性

## （2）请结合上课时所讲的知识阐述 DESeq2 和 edgeR 中如何对数据进行 normalization，列出并解释具体的公式。

### DESeq2 的 Normalization：RLE (Relative Log Expression) 方法（其底层逻辑即为 Median of Ratios 中位数比值法），通过 `estimateSizeFactors()` 函数实现。

- **Step 1：计算每个基因( $g$ )的原始 count 在所有样本中的几何平均值**
  
$$
\text{Geometric Mean}_g = \left(\prod_{l=1}^{N} K_{gl}\right)^{1/N}
$$

- **Step 2：将每个基因( $g$ )的 count 数除以其对应的几何平均数**

$$
\text{Ratio}_{gj} = \frac{K_{gj}}{\left(\prod_{l=1}^{N} K_{gl}\right)^{1/N}}
$$

- **Step 3：计算每个样本( $j$ )的归一化因子（Normalization factor, $C_j$）**
  
    取每个样本所有基因上述比值的中位数（median），作为该样本的归一化因子：

$$
C_j = \text{median}_g \left\lbrace \frac{K_{gj}}{\left( \prod_{l=1}^{N} K_{gl} \right)^{1/N}} \right\rbrace
$$

- **Step 4：用归一化因子（ $C_j$ ）对原始 count 进行归一化**

$$
\frac{C_{raw}(g)}{C_j} = \frac{K_{gj}}{C_j}
$$

  > 为什么用中位数而不是均值？  
    使用**中位数**而非均值，是为了对少数极高表达基因（如管家基因）的影响具有鲁棒性。这些基因的比值可能极大，但中位数不受极端值影响。

- DESeq2 的统计模型

  归一化后，DESeq2 对每个基因 $g$ 在样本 $j$ 中的 count 建立负二项分布模型：

    $$K_{gj} \sim \text{NB}(\mu_{gj}, \alpha_g)$$

    $$\mu_{gj} = C_j \cdot q_{gj}$$

    $$\ln(q_{gj}) = \beta_{g0} + \beta_{g1} \cdot x_j$$

    其中：
    - $\mu_{gj}$：期望 count
    - $\alpha_g$：基因 $g$ 的离散度参数（dispersion）
    - $C_j$：样本 $j$ 的归一化因子（Normalization factor）
    - $q_{gj}$：归一化后的期望表达量
    - $\beta_{g0}$：截距（基础表达水平）
    - $\beta_{g1}$：处理效应（即 log2FoldChange）
    - $x_j$：样本 $j$ 的处理条件（0 = Control，1 = Treatment）

### edgeR 的 Normalization：TMM (Trimmed Mean of M-values) 方法，通过代码中的 `calcNormFactors()` 函数实现。

- **Step 1：初步预处理**  
  通过对每个样本的测序文库大小（sequencing library size）进行标准化来计算 $K_{gj}$，并移除未表达的基因。

- **Step 2：选择参考样本 ( $r$ )**  
  选择一个参考样本 ( $r$ )，该样本与每个样本中所有基因的 75% 分位数均值差异最小。

- **Step 3：筛选代表性基因集 ( $G$ ) 并进行截尾 (Trimming)**  
  计算 M 值（对数表达比值）和 A 值（绝对表达量），并切除极端数据（切除 M 值上下各 30% 和 A 值上下各 5% 的数据），得到代表性基因集 $G$。

$$
M_g(j, r) = \log_2(K_{gj}/D_j) - \log_2(K_{gr}/D_r)
$$

$$
A_g(j, r) = \frac{1}{2}(\log_2(K_{gj}/D_j) + \log_2(K_{gr}/D_r))
$$

  其中：
  - $K_{gj}$（也称 $C_{raw}(g)$）：样本 $j$ 中比对到基因 $g$ 上的 counts/reads 数
  - $D_j$：样本 $j$ 的总 reads 数（即 library size）
  - $r$：参考样本
  - $G$：截尾后保留的代表性基因集

- **Step 4：计算代表性基因集 ( $G$ ) 中每个基因( $g$ )的权重**
  
$$
w_g(j, r) = \left( \frac{D_j - K_{gj}}{D_j K_{gj}} + \frac{D_r - K_{gr}}{D_r K_{gr}} \right)^{-1}
$$

- **Step 5：计算每个样本 ($j$ )的归一化因子 ( $C_j$ )**  
  利用加权均值计算 TMM 值，再求 2 的指数得到样本 $j$ 的归一化因子 $C_j$：  

$$
\text{TMM}(j, r) = \frac{\sum_{g \in G^{\ast}} w_g(j, r) M_g(j, r)}{\sum_{g \in G^{\ast}} w_g(j, r)}
$$

$$
C_j = 2^{\text{TMM}(j, r)}
$$

- **Step 6：对每个基因的 reads/counts 进行归一化**
  
$$
\frac{C_{raw}(g)}{C_j} = \frac{K_{gj}}{C_j}
$$

- edgeR 的统计模型

  归一化后，edgeR 对每个基因 $g$ 在样本 $j$ 中的 count 建立负二项分布模型：

  $$K_{gj} \sim \text{NB}(\mu_{gj}, \phi_g)$$

  $$\mu_{gj} = D_j \times C_j \times \lambda_{gj}$$

  $$\ln(\lambda_{gj}) = \mathbf{x}_j^T \boldsymbol{\beta}_g$$

  其中：
    - $K_{gj}$：原始 count
    - $\phi_g$：基因 $g$ 的离散度（dispersion）
    - $D_j$：样本 $j$ 的总 reads 数（Library size）
    - $C_j$：样本 $j$ 的归一化因子（Normalization factor）
    - $\lambda_{gj}$：归一化后的期望表达比例
    - $\mathbf{x}_j$：样本 $j$ 的设计向量
    - $\boldsymbol{\beta}_g$：回归系数向量（包含 log2FoldChange）

### DESeq2 vs edgeR Normalization 对比

| 特性 | DESeq2（Median of Ratios） | edgeR（TMM） |
|------|---------------------------|--------------|
| **核心思想** | 用几何平均作为参考，取比值中位数 | 用截尾均值消除极端基因影响 |
| **鲁棒性** | 对极端高表达基因鲁棒（中位数） | 对极端高表达基因鲁棒（截尾） |
| **假设** | 大多数基因不差异表达 | 大多数基因不差异表达 |
| **输出** | size factor（每个样本一个值） | normalization factor（每个样本一个值） |
| **实际效果** | 两种方法结果通常非常接近 | 两种方法结果通常非常接近 |

## （3）利用以上介绍的方法和数据，分别使用 DESeq2 和 edgeR 找出 uvr8 突变型（uvr8\*）在光照前后的差异基因，保存为文本文件

### 使用 DESeq2 分析 uvr8\* 差异基因

```r
library(DESeq2)

# Step 1: 读取原始 count 矩阵
raw.counts <- read.table("count_exon.txt", sep='\t', header = T, row.names = 1)

# Step 2: 提取 uvr8 突变型数据（UD0 代表光照前 Control，UD1 代表光照后 Treatment）
uvr8.raw.counts <- raw.counts[, c("UD0_1", "UD0_2", "UD0_3", "UD1_1", "UD1_2", "UD1_3")]

# Step 3: 过滤低表达基因
uvr8.filtered.counts <- uvr8.raw.counts[rowMeans(uvr8.raw.counts) > 5, ]

# Step 4: 定义样本条件
# Control = 光照前（dark），Treatment = 光照后（light）
conditions <- factor(c(rep("Control", 3), rep("Treatment", 3)), levels = c("Control", "Treatment"))
colData <- data.frame(row.names = colnames(uvr8.filtered.counts), conditions = conditions)

# Step 5: 创建 DESeqDataSet 并进行差异分析
dds <- DESeqDataSetFromMatrix(uvr8.filtered.counts, colData, design = ~conditions)
dds <- DESeq(dds)
res <- results(dds)

# Step 6: 保存所有基因结果
write.table(res, "deseq2.uvr8.light.vs.dark.all.txt", sep='\t', row.names = T, quote = F)

# Step 7: 筛选差异显著基因（|log2FC| > 1, padj < 0.05）
diff.deseq2 <- subset(res, padj < 0.05 & abs(log2FoldChange) > 1)
write.table(diff.deseq2, "deseq2.uvr8.light.vs.dark.sig.txt", sep='\t', row.names = T, quote = F)

cat("DESeq2 找到的差异基因数：", nrow(diff.deseq2), "\n")
```

输出：
```
DESeq2 找到的差异基因数： 95
```

### 使用 edgeR 分析 uvr8\* 差异基因

```r
library(edgeR)

# Step 1: 读取数据
raw.counts <- read.table("count_exon.txt", sep='\t', header = T, row.names = 1)

# Step 2: 提取 uvr8 突变型数据（UD0 代表光照前 Control，UD1 代表光照后 Treatment）
uvr8.raw.counts <- raw.counts[, c("UD0_1", "UD0_2", "UD0_3", "UD1_1", "UD1_2", "UD1_3")]

# Step 3: 过滤低表达基因
uvr8.filtered.counts <- uvr8.raw.counts[rowMeans(uvr8.raw.counts) > 5, ]

# Step 4: 定义样本条件和 design 矩阵
conditions <- factor(c(rep("Control", 3), rep("Treatment", 3)), levels = c("Control", "Treatment"))
design <- model.matrix(~conditions)

# Step 5: 创建 DGEList 对象并进行 TMM 标准化
y <- DGEList(counts = uvr8.filtered.counts)
y <- calcNormFactors(y, method = "TMM")

# Step 6: 估计离散度并拟合 GLM
y <- estimateDisp(y, design = design)
fit <- glmFit(y, design = design)

# Step 7: 似然比检验
lrt <- glmLRT(fit, coef = 2)

# Step 8: 获取所有基因结果并保存
diff.table <- topTags(lrt, n = nrow(y))$table
write.table(diff.table, file = 'edger.uvr8.light.vs.dark.all.txt', sep = "\t", quote = F, row.names = T, col.names = T)

# Step 9: 筛选差异显著基因（|log2FC| > 1, FDR < 0.05）
diff.edger <- diff.table[abs(diff.table$logFC) > 1 & diff.table$FDR < 0.05, ]
write.table(diff.edger, file = 'edger.uvr8.light.vs.dark.sig.txt', sep = "\t", quote = F, row.names = T, col.names = T)

cat("edgeR 找到的差异基因数：", nrow(diff.edger), "\n")
```

输出：
```
edgeR 找到的差异基因数： 67
```

## （5）对于 edgeR 找出的 FDR < 0.05 的基因，选出 log2FoldChange 最大的 10 个基因和最小的 10 个基因。计算原始表达量矩阵的 log10CPM 值并对每个基因进行 Z-score 处理，使用筛选出来的 20 个基因绘制热图（heatmap）作为最后结果输出。

```r
library(edgeR)
library(pheatmap)

# 1. 筛选显著基因并按 logFC 排序
sig.genes <- diff.table[diff.table$FDR < 0.05, ]
sig.genes <- sig.genes[order(sig.genes$logFC, decreasing = TRUE), ]

# 2. 提取两端 Top 10 基因名（前 10 个上调，后 10 个下调）
top20.genes <- rownames(sig.genes)[c(1:10, (nrow(sig.genes)-9):nrow(sig.genes))]

# 3. 计算 log10(CPM+1) 并提取这 20 个基因
log10cpm.top20 <- log10(cpm(y)[top20.genes, ] + 1)

zscore.top20 <- t(scale(t(log10cpm.top20)))
```
