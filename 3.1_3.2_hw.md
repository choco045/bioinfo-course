## 请问上面的例子中，Fold Enrichment和P value是如何计算的？请写出公式，并解释原理。此外，在定义显著富集的GO terms时为什么一般不是参考P value的大小，而是要计算一个FDR来作为参考？

**解答：**

### Fold Enrichment 的计算

**公式：**

$$\text{Fold Enrichment} = \frac{k / n}{K / N}= \frac{k}{n} \div \frac{K}{N} = \frac{k \cdot N}{n \cdot K}$$

其中：
- $N$：参考基因组中所有被 GO 注释的基因总数（例子中 N = 21042）
- $n$：用户上传基因中成功映射到背景的基因数（例子中 n = 50）
- $K$：参考基因组中被注释到某一 GO term 的基因数（例子中 K = 208，即 DNA replication）
- $k$：用户上传基因中被注释到该 GO term 的基因数（例子中 k = 6）

**以例子验证：**

$$\text{expected} = \frac{K \times n}{N} = \frac{208 \times 50}{21042} \approx 0.4942$$

$$\text{Fold Enrichment} = \frac{k}{\text{expected}} = \frac{6}{0.4942} \approx 12.14$$

**原理：** Fold Enrichment 衡量的是"实际观测到的富集程度"相对于"随机期望值"的倍数。若 Fold Enrichment > 1，说明该 GO term 在用户基因集中比随机情况下更多地出现，即存在富集；倍数越大，富集程度越高。

---

### P value 的计算

**使用超几何分布（Hypergeometric Test）：**

$$P = \sum_{i=k}^{\min(n,K)} \frac{\binom{K}{i} \binom{N-K}{n-i}}{\binom{N}{n}}$$

**原理：** 超几何分布描述的是"不放回抽样"问题。将参考基因组中的 $N$ 个基因看作总体，其中 $K$ 个属于某 GO term；从中随机抽取 $n$ 个基因（即用户上传的基因集），问恰好有 $k$ 个或更多属于该 GO term 的概率是多少。

- 若该概率（P value）很小，说明用户基因集中出现如此多该 GO term 基因并非偶然，即该 GO term 在用户基因集中显著富集。
- P value 越小，富集越显著。

---

### 为什么要用 FDR 而不是直接用 P value？

**多重检验问题（Multiple Testing Problem）：**

在 GO 富集分析中，我们同时对**数千个 GO terms** 进行统计检验。即使每个检验的显著性阈值设为 $\alpha = 0.05$，在没有任何真实富集的情况下，平均也会有 $5\%$ 的 GO terms 被错误地判定为显著（即假阳性）。

例如，若同时检验 1000 个 GO terms，即使所有 GO terms 都不富集，也预期会有约 $1000 \times 0.05 = 50$ 个 GO terms 的 P value < 0.05，这些都是假阳性结果。

**FDR（False Discovery Rate，假发现率）的意义：**

FDR 是对多重检验进行校正的方法，常用 **Benjamini-Hochberg（BH）方法** 计算。FDR 控制的是"在所有被判定为显著的结果中，假阳性所占的比例"。

- 使用 FDR < 0.05 作为阈值，意味着在所有被认为显著富集的 GO terms 中，最多允许 5% 是假阳性
- 相比直接使用 P value，FDR 更加严格，能有效控制假阳性率，使结果更可靠

**结论：** 在进行大规模多重检验时（如 GO 富集分析），必须使用 FDR 而非原始 P value 作为显著性判断标准，以避免大量假阳性结果干扰生物学解读。

## 请用KEGG enrichment分析上一章（GO enrichment analysis）中的基因集，比较两章的结果，总结两种方法得到的生物学意义有哪些异同

**解答：**

### 分析步骤

使用与 GO 分析相同的基因集（`KEGG_gene.txt`，即 51 个人类基因的 Ensembl ID），在 [DAVID](https://david.ncifcrf.gov/) 平台上进行 KEGG 富集分析：

1. 访问 DAVID 网站（https://david.ncifcrf.gov/）
2. 将基因 Ensembl ID 粘贴到 "Enter Gene List" 文本框
3. 选择 ID 类型为 `ENSEMBL_GENE_ID`
4. 选择列表类型为 `Gene List`
5. 点击 `Submit List` 提交
6. 在结果页面查看 `KEGG_PATHWAY` 类别的富集结果

### GO 分析 vs KEGG 分析的异同

**相同点：**

| 方面 | 说明 |
|------|------|
| **统计方法** | 两者均使用超几何分布（Hypergeometric Test）进行富集检验 |
| **显著性判断** | 均以 FDR < 0.05 作为显著富集的阈值 |
| **基因集** | 使用相同的基因列表进行分析 |
| **核心逻辑** | 均通过比较"观测值"与"期望值"来判断富集程度 |

**不同点：**

| 比较维度 | GO 分析 | KEGG 分析 |
|---------|---------|----------|
| **注释数据库** | Gene Ontology 数据库 | KEGG Pathway 数据库 |
| **注释层次** | 三个维度：分子功能(MF)、生物过程(BP)、细胞组分(CC) | 代谢通路、信号通路、疾病通路等 |
| **结果粒度** | 功能描述较为宽泛，层次丰富（从宏观到微观） | 通路描述具体，有明确的生化反应图 |
| **生物学意义** | 揭示基因的功能类别（"这些基因做什么"） | 揭示基因参与的具体通路（"这些基因在哪条通路中起作用"） |
| **可视化** | 以 GO term 列表或 DAG 图展示 | 可在 KEGG 通路图上高亮显示富集基因 |
| **覆盖范围** | 覆盖更广，几乎所有基因都有 GO 注释 | 覆盖范围相对较窄，部分基因无 KEGG 通路注释 |

### 生物学意义总结

- **GO 分析**侧重于描述基因集的**功能属性**，能从分子功能、生物过程、细胞定位三个角度全面刻画基因集的特征，适合初步了解基因集的整体功能倾向。例如，若 GO 分析显示"DNA replication"显著富集，说明该基因集整体上与 DNA 复制功能相关。

- **KEGG 分析**侧重于揭示基因集参与的**具体生物学通路**，结果更具机制性和可解释性。例如，KEGG 分析可能显示基因集富集于"Cell cycle"（细胞周期）通路，并能在通路图上直观看到哪些步骤被激活。

- **两者互补**：GO 分析提供功能分类的全景视图，KEGG 分析提供通路层面的机制解析。在实际研究中，通常同时进行两种分析，以获得更全面的生物学解读。对于同一基因集，GO 和 KEGG 分析结果往往在生物学主题上一致（如都指向细胞增殖相关功能），但 KEGG 能提供更具体的通路信息，而 GO 能提供更丰富的功能层次描述。
