## 在GenomicFeatures package中，默认情况下promoters函数是如何定义promoter对应的genomic intervals的？

在 `GenomicFeatures` 包中，`promoters()` 函数默认将 **转录起始位点（TSS）上游 2000 bp、下游 200 bp** 的区间定义为启动子（promoter）对应的基因组区间。

具体来说，对于正链（+）基因，promoter 区间为 `[TSS - 2000, TSS + 200)`；对于负链（-）基因，方向相反，同样以 TSS 为中心向两侧延伸。

函数签名为：
```r
promoters(x, upstream=2000, downstream=200, ...)
```

其中 `upstream` 和 `downstream` 参数均可由用户自定义修改。

## meme的-mod参数有oops，zoops，anr三个可选项，阅读文档回答，它们分别是什么含义？

MEME 的 `-mod` 参数用于指定 motif 在输入序列中的出现模型（occurrence model），三个选项含义如下：

| 参数值 | 全称 | 含义 |
|--------|------|------|
| **oops** | One Occurrence Per Sequence | 假设每条输入序列中**恰好出现一次**该 motif。这是最严格的假设，适用于已知每条序列都含有该 motif 的情况（如 ChIP-seq peaks）。统计功效最高，但若假设不成立则结果不可靠。 |
| **zoops** | Zero or One Occurrence Per Sequence | 假设每条输入序列中出现**零次或一次**该 motif。比 oops 更宽松，允许部分序列不含该 motif，是最常用的默认模式，适合大多数实际场景。 |
| **anr** | Any Number of Repetitions | 假设每条输入序列中可以出现**任意次数**的该 motif（包括零次）。最宽松的假设，适用于 motif 可能在序列中多次重复出现的情况（如串联重复序列）。计算量最大。 |

## 在5'UTR序列上重复motif discovery分析，对known-motifs/Ray2013_rbp_Homo_sapiens.meme中提供的motif在5'UTR序列上进行motif enrichment分析

### Motif Discovery（在 5' UTR 上）

首先需要过滤掉长度过短的序列（MEME 要求每条输入序列长度必须大于指定的 motif 宽度）：

```bash
# 使用 awk 过滤掉长度小于 10 bp 的序列（对应 -minw 6 的要求，保守起见过滤 <10bp）
# 先将多行 FASTA 转为单行，再过滤，再转回多行格式
awk '/^>/{header=$0; next} {if(length($0)>=10) print header"\n"$0}' \
    sequences/five.prime.utrs.spliced.fa > sequences/five.prime.utrs.spliced.filtered.fa

# 在 5' UTR 序列上运行 MEME 进行 de novo motif discovery
# -dna: 输入为 DNA 序列
# -minw 6 -maxw 10: motif 宽度范围 6~10 bp
# -oc: 输出目录
# -nmotifs 5: 最多发现 5 个 motif
meme -dna -minw 6 -maxw 10 -oc five_utr.motif.discovery -nmotifs 5 \
    sequences/five.prime.utrs.spliced.filtered.fa
```

### Motif Enrichment（在 5' UTR 上，使用 Ray2013 RBP motif 库）

```bash
# 使用 AME 对 Ray2013 RBP motif 库在 5' UTR 序列上进行富集分析
# --control: 使用 dinucleotide-shuffled 序列作为对照
# --oc: 输出目录
# 前景序列：5' UTR spliced 序列
# motif 库：Ray2013 人类 RBP motif 数据库
ame --control sequences/five.prime.utrs.spliced.dinuc.shuffled.fa \
    --oc five_utr.motif.enrichment \
    sequences/five.prime.utrs.spliced.filtered.fa \
    known-motifs/Ray2013_rbp_Homo_sapiens.meme
```

**分析意义**：Ray2013 数据库收录了大量人类 RNA 结合蛋白（RBP）的结合 motif。在 5' UTR 上进行富集分析，可以发现哪些 RBP 倾向于结合目标基因的 5' UTR，从而推断可能参与翻译调控的 RBP。

## 对我们在CHIP-seq一节的作业中获得的peaks对应的序列，用meme进行motif finding的分析

### 步骤一：用 HOMER 提取 ChIP-seq peak 对应的序列

```bash
# homerTools extract: 从基因组中提取 peak 区域对应的 FASTA 序列
# CHIP.peak: homer findPeaks 输出的 peak 文件
# /home/test/homer/data/genomes/sacCer2/: 酵母基因组目录
# -fa: 输出为 FASTA 格式
homerTools extract CHIP.peak /home/test/homer/data/genomes/sacCer2/ -fa > chip_peaks.fa
```

### 步骤二：用 MEME 进行 Motif Finding

```bash
# 在 ChIP-seq peak 序列上运行 MEME 进行 motif discovery
# -dna: DNA 序列
# -mod zoops: 每条序列出现零次或一次（适合 ChIP-seq 数据）
# -minw 6 -maxw 20: motif 宽度范围（ChIP-seq 转录因子结合位点通常 6~20 bp）
# -nmotifs 5: 最多发现 5 个 motif
# -oc: 输出目录
meme chip_peaks.fa -dna -mod zoops -minw 6 -maxw 20 \
    -nmotifs 5 -oc chip_peaks.motif.discovery
```

### 步骤三：结果解读

MEME 输出目录中包含：
- `meme.html`：可视化结果，展示发现的 motif 的 sequence logo、E-value 等统计信息
- `meme.txt`：文本格式的详细结果
- `meme.xml`：XML 格式结果，可供下游工具使用

通过比对发现的 motif 与已知转录因子数据库（如 JASPAR、TRANSFAC），可以推断 ChIP-seq 实验中富集的转录因子的结合序列偏好性。

## 阅读2014,NAR,《A novel approach to represent and compare RNA secondary structures》这篇文章，简要描述BEAR encoder这个工具是怎样把二维结构编码到一维序列上的

**BEAR（BEar Annotated RNA）encoder** 的核心思想是将 RNA 二级结构的点括号表示（dot-bracket notation）转化为一套新的字母表编码，从而将二维结构信息压缩到一维序列中。

具体编码规则如下：

1. **结构元件分类**：BEAR 将 RNA 二级结构中的每个核苷酸位置按其所处的结构元件分为以下几类：
   - **Stem（茎）**：碱基配对区域，用字母 `Z`、`A`、`Q` 表示（根据茎的长度细分）
   - **Loop（发夹环）**：茎末端的单链环，用 `X`、`S`、`W` 表示
   - **InternalLoop（内部环）**：两段茎之间的不配对区域，用 `C`、`D`、`E` 表示
   - **Bulge（凸起）**：一侧有不配对核苷酸的结构，用 `B` 表示
   - **BulgeBranch（凸起分支）**：凸起处的分支，用 `G` 表示
   - **Branching（多分支环）**：三条或以上茎汇聚的区域，用 `T` 表示
   - **InternalLoopBranch（内部环分支）**：内部环处的分支，用 `N`、`H`、`Y` 表示

2. **编码过程**：
   - 输入：RNA 序列 + RNAfold 预测的点括号结构（dot-bracket notation）
   - 对每个核苷酸位置，根据其在二级结构中所属的结构元件类型，将其替换为对应的 BEAR 字母
   - 输出：与原序列等长的 BEAR 编码字符串

3. **优势**：编码后的 BEAR 序列与原核苷酸序列等长，可以直接套用成熟的 sequence motif 发现算法（如 MEME）来发现 structure motif，大大降低了方法开发的复杂度。

4. **局限性**：由于将二维结构信息强制压缩为一维字符串，不可避免地丢失了部分空间结构信息（如茎的具体配对关系、结构的整体拓扑等）。

## RNApromo这个工具是2008,PNAS,《Computational prediction of RNA structural motifs involved in posttranscriptional regulatory processes》这篇文章发表的。阅读文章，简要回答RNApromo和meme的区别在什么地方？

**RNApromo** 与 **MEME** 的主要区别如下：

| 比较维度 | MEME | RNApromo |
|---------|------|----------|
| **分析对象** | 纯序列（sequence motif） | 序列 + 二级结构（structure motif） |
| **输入** | 核苷酸序列（FASTA） | 核苷酸序列（同时考虑其二级结构） |
| **motif 表示** | PWM（位置权重矩阵），仅描述序列偏好性 | SCFG（随机上下文无关文法），同时描述序列和结构偏好性 |
| **算法基础** | EM（期望最大化）算法 | 基于 SCFG 的概率模型，结合结构预测 |
| **结构信息** | 不考虑 RNA 二级结构 | 显式建模 RNA 二级结构，能发现具有特定折叠模式的 motif |
| **适用场景** | 转录因子结合位点、DNA/RNA 序列 motif | RNA 结合蛋白识别位点、具有保守二级结构的 RNA 调控元件 |
| **计算复杂度** | 相对较低 | 较高（需要同时考虑序列和结构空间） |

**核心区别总结**：MEME 只在序列层面寻找 motif，而 RNApromo 能够同时在序列和结构层面寻找 motif。RNApromo 使用 SCFG 模型，可以捕捉 RNA 二级结构中的碱基配对关系和结构拓扑，因此更适合发现那些功能依赖于特定空间折叠结构的 RNA 调控元件（如 IRES、miRNA 结合位点等）。
