## 1. 请解释在ChIP-seq实验中为什么一般都要平行做一个 control（通常叫 input）的实验。

在 ChIP-seq 实验中，IP 组通过特异性抗体对目标蛋白（或组蛋白修饰）结合的 DNA 片段进行富集，但富集过程中不可避免地存在以下噪声来源：

1. **非特异性抗体结合**：抗体可能与非目标蛋白发生非特异性结合，导致某些基因组区域被假性富集。
2. **基因组可及性偏差**：基因组中不同区域的染色质开放程度不同，开放区域（如启动子、增强子）天然更容易被超声打断并进入文库，即使没有目标蛋白结合，这些区域的 reads 也会偏多。
3. **PCR 扩增偏差**：文库制备过程中 PCR 扩增会对 GC 含量不同的区域产生不均一的扩增效率，造成系统性偏差。
4. **测序偏差**：测序仪对不同序列组成的片段存在固有的测序偏好。

**Input 对照**（即不经过免疫沉淀、直接对打断后的染色质 DNA 建库测序）代表了基因组的"背景"信号，反映了上述所有非特异性因素的综合影响。

通过将 IP 组的信号与 Input 组的背景信号进行比较（如计算 Fold Change、泊松检验等），可以：
- 排除基因组可及性偏差和测序偏差的干扰；
- 识别真正由目标蛋白结合引起的富集区域（peaks）；
- 降低假阳性率，提高 peak calling 的特异性和可靠性。

因此，Input 对照是 ChIP-seq 实验设计中不可缺少的质控手段，是区分真实信号与背景噪声的关键参照。

## 2. 请解释 `findPeaks` 和 `findMotifsGenome.pl` 主要参数的含义。

### `findPeaks` 主要参数

`findPeaks` 是 HOMER 中用于 peak calling 的核心命令，基本用法：

```bash
findPeaks <IP tag directory> -style <factor|histone> -o <output peak file> -i <Input tag directory>
```

| 参数 | 含义 |
|------|------|
| `<IP tag directory>` | 第一个位置参数，IP 实验样本的 tag directory 路径（由 `makeTagDirectory` 生成） |
| `-style factor` | Peak 类型为**转录因子**模式：寻找较窄的 narrow peaks，要求 IP 信号相对于 input 和局部背景均有显著富集（≥4 倍，泊松 p < 0.0001） |
| `-style histone` | Peak 类型为**组蛋白修饰**模式：寻找较宽的 broad peaks，适用于 H3K4me3、H3K27me3 等组蛋白修饰 ChIP-seq |
| `-o <file>` | 输出 peak 文件的路径，结果以制表符分隔的文本格式保存 |
| `-i <Input tag directory>` | Input（对照）样本的 tag directory 路径，用于计算 Fold Change vs Control 和相应 p 值 |
| `-C <#>` | Clonal Fold Change 阈值（默认 2），用于过滤可能来自重复序列扩增的假阳性 peak |

**Peak 显著性判断标准（factor 模式）**：
- IP 与 Input 的标准化 tag 数之比（Fold Change vs Control）≥ 4
- 泊松检验 p-value vs Control < 0.0001
- Peak 处 tag 密度与周围 10 kb 区域之比（Fold Change vs Local）≥ 4
- 泊松检验 p-value vs Local < 0.0001

### `findMotifsGenome.pl` 主要参数

`findMotifsGenome.pl` 是 HOMER 中用于 motif 发现的 Perl 脚本，基本用法：

```bash
findMotifsGenome.pl <peak file> <genome> <output directory> [options]
```

| 参数 | 含义 |
|------|------|
| `<peak file>` | 第一个位置参数，peak calling 的结果文件（如 `findPeaks` 输出的 peak 文件） |
| `<genome>` | 第二个位置参数，参考基因组名称（如 `sacCer2` 为酵母，`hg38` 为人类基因组），HOMER 据此提取 peak 区域的序列 |
| `<output directory>` | 第三个位置参数，motif 分析结果的输出目录 |
| `-len <#>` | Motif 长度（bp），默认为 8,10,12；可用逗号分隔多个长度（如 `-len 8,10,12`）。转录因子 motif 通常为 6–12 bp |
| `-size <#>` | 用于 motif 发现的 peak 区域大小（默认 200 bp）。转录因子推荐 50 bp（主要 motif）或 200 bp（主要 + 协同 motif）；组蛋白修饰推荐 500–1000 bp |
| `-S <#>` | 每个长度要发现的 motif 数量（默认 25）。25 个已经相当多，一般不需要增大 |
| `-p <#>` | 并行计算使用的 CPU 核心数，可加速分析 |
| `-bg <file>` | 自定义背景序列文件（默认 HOMER 自动生成随机背景） |

**主要输出**：
- `homerResults.html`：de novo motif 发现结果，包含每个 motif 的序列 logo、富集 p 值、与已知 motif 的相似性等信息。
- `knownResults.html`：已知 motif 数据库中各 motif 在 peak 区域的富集情况。
- `knownResults.txt`：已知 motif 富集统计的文本格式，可用 Excel 打开。

## 3. 使用 homer 对酵母 Snf1 蛋白 ChIP-seq 数据进行 peak calling 和 motif finding 分析，提交找到的 motif 截图，以及 Fold Change (vs Control) ≥ 8 且 p-value (vs Control) < 10⁻⁸ 的 peaks。

### 分析步骤

数据位于容器的 `/home/test/chip-seq/homework` 目录，`ip.chrom_part.bam` 为 IP 实验数据，`input.chrom_part.bam` 为背景数据。

#### 步骤一：生成 Tag Directory

```bash
# 进入 homework 工作目录
cd /home/test/chip-seq/homework

# 为 IP 样本（Snf1 ChIP）生成 tag directory
makeTagDirectory ip_tags ip.chrom_part.bam

# 为 Input（对照）样本生成 tag directory
makeTagDirectory input_tags input.chrom_part.bam
```

#### 步骤二：Peak Calling

```bash
# 使用 findPeaks 进行 peak calling（转录因子模式）
# Snf1 是激酶/转录调控因子，使用 factor 模式（narrow peaks）
findPeaks ip_tags/ -style factor -o snf1_peaks.txt -i input_tags/
```

#### 步骤三：筛选高置信度 peaks

```bash
# 筛选 Fold Change vs Control >= 8 且 p-value vs Control < 1e-8 的 peaks
# 第11列为 Fold Change vs Control，第12列为 p-value vs Control
# 跳过注释行（以 # 开头）
awk '!/^#/ && $11 >= 8 && $12 < 1e-8' snf1_peaks.txt > snf1_peaks_filtered.txt

# 查看筛选结果数量
wc -l snf1_peaks_filtered.txt
```

#### 步骤四：Motif Analysis

```bash
# 对筛选后的 peaks 进行 motif 发现
# sacCer2：酵母参考基因组
# -len 8,10,12：分析 8、10、12 bp 长度的 motif
findMotifsGenome.pl snf1_peaks_filtered.txt sacCer2 snf1_motif_output/ -len 8,10,12
```

### 分析结果说明

**Peak Calling 结果**：

根据 HOMER 的 peak calling 标准（factor 模式），筛选条件为：
- **Fold Change vs Control ≥ 8**：IP 组信号至少是 Input 背景的 8 倍，表明该区域有强烈的 Snf1 蛋白富集。
- **p-value vs Control < 10⁻⁸**：泊松检验高度显著，排除随机波动的可能性极高。

满足上述条件的 peaks 代表 Snf1 蛋白在酵母基因组上最可靠的结合位点。

**Motif Analysis 结果**：

`homerResults.html` 中展示的 de novo motif 代表了 Snf1 结合位点处富集的序列特征。Snf1 是酵母中 AMP 激活蛋白激酶（AMPK）的同源物，参与碳源代谢调控。其结合的转录因子（如 Mig1、Cat8 等）具有特定的 DNA 结合 motif，预期在 motif 分析中可以发现与这些转录因子相关的序列模式。
